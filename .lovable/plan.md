

# تعديل صلاحيات المشرف - تحديد الورشة

## الفكرة
عند تعيين شخص كـ"مشرف"، يجب تحديد رقم الورشة (أولى - خامسة) التي يشرف عليها. المشرف يرى فقط أسماء الأشخاص في ورشته المحددة.

## الخطوات

### 1. تعديل قاعدة البيانات
- إضافة عمود `supervised_workshop` (text, nullable) إلى جدول `user_roles`
- يحفظ قيمة مثل "ورشة أولى"، "ورشة ثانية"، إلخ

### 2. تعديل `src/hooks/useAuth.tsx`
- إضافة `supervisedWorkshop: string | null` إلى `AuthContextType`
- جلب قيمة `supervised_workshop` مع دور المستخدم من جدول `user_roles`
- تمريرها عبر Context لجميع الصفحات

### 3. تعديل `src/pages/AdminPage.tsx`
- عند اختيار دور "مشرف" لمستخدم، يظهر خيارات اختيار الورشة (أولى - خامسة)
- حفظ رقم الورشة في عمود `supervised_workshop` مع الدور
- عرض رقم الورشة بجانب اسم المشرف في القائمة

### 4. تعديل `src/pages/AttendancePage.tsx`
- إذا كان المستخدم الحالي مشرف (`userRole === "supervisor"`):
  - في قسم الورشة: يعرض فقط الأشخاص الذين `workshop_number` يطابق `supervisedWorkshop`
  - يمنع الوصول لباقي الورش

## التفاصيل التقنية

```sql
ALTER TABLE user_roles ADD COLUMN supervised_workshop text;
```
